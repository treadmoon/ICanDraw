# ICanDraw — AI 驱动的数据可视化智能画布

## 产品定位

用自然语言对话，AI 自动生成数据可视化故事板——融合交互式图表与手绘风格图解的智能画布。

核心逻辑：用户说话/打字 → AI 理解意图 → 自动生成"图表 + 图解 + 文字注释"的组合画面。

用户不需要懂 ECharts 配置，不需要会画图，AI 全权负责内容生成和排版。

## 目标用户

- 产品经理
- 数据分析师
- 技术架构师
- 创业团队

## 核心使用场景

### 场景 1：数据分析叙事

> 用户："帮我分析这份销售数据，做一个季度汇报的可视化"
>
> AI：自动解析数据 → 生成关键指标卡片 + 趋势折线图 + 手绘风格的原因标注箭头 + 文字总结

### 场景 2：概念解释

> 用户："用图解释微服务架构和单体架构的区别"
>
> AI：左右对比布局，手绘架构图 + 性能对比柱状图 + 关键差异标注

### 场景 3：商业计划可视化

> 用户："我要做一个咖啡店创业计划的展示"
>
> AI：市场规模饼图 + 成本结构图 + 增长预测折线图 + 手绘商业模式画布

### 场景 4：实时数据看板

> 用户："监控我们网站的实时流量，重点关注转化率"
>
> AI：自动连接数据源 → 生成实时仪表盘 → 异常时自动标注手绘风格的警告

## AI 驱动的交互模式

### 1. 对话式创作（主交互）

- 左侧对话面板，右侧实时画布
- 用户每说一句话，画布实时更新
- 支持追问和迭代："把柱状图换成折线图"、"突出 Q3 的数据"、"加个对比"

### 2. 数据投喂

- 拖入 CSV/Excel/JSON → AI 自动分析并生成可视化建议
- 粘贴 URL → AI 抓取数据并可视化
- 连接 API → AI 持续监控并自动更新

### 3. AI 主动洞察

- AI 不只是被动执行，还会主动发现数据中的异常、趋势、关联
- "我注意到你的用户留存率在 3 月有明显下降，要我深入分析吗？"

### 4. 风格控制

- "用更正式的风格" → 切换为商务图表风格
- "随意一点" → 手绘涂鸦风格
- "做成演示文稿" → 自动分页排版

## 技术方案

```
前端：React + Excalidraw SDK + ECharts
AI 层：LLM (GPT/Claude API) + Function Calling
  ├── 数据分析 Agent（pandas/数据处理）
  ├── 图表生成 Agent（输出 ECharts option JSON）
  ├── 图解生成 Agent（输出 Excalidraw elements JSON）
  └── 布局 Agent（组合排版）
后端：Node.js / Python
  ├── 对话管理
  ├── 数据源连接器
  └── 文件解析（CSV/Excel/JSON）
存储：用户画布数据 + 对话历史 + 数据缓存
```

关键技术决策：

- AI 输出的是结构化 JSON（ECharts option + Excalidraw elements），不是图片
- 用户可以继续编辑、AI 可以增量修改
- 用 Function Calling / Tool Use 让 LLM 调用图表生成和图解生成工具

## 分阶段路线

### Phase 1 — AI 图表生成器（4-6 周）

- 对话界面 + 画布
- 用户输入自然语言 → AI 生成 ECharts 图表
- 支持上传 CSV 数据
- 支持对话迭代修改图表

### Phase 2 — 混合画布（4 周）

- AI 同时生成图表 + 手绘图解 + 文字注释
- 自动布局排版
- 用户可手动微调 AI 生成的内容

### Phase 3 — 数据智能（4 周）

- 数据源连接
- AI 主动洞察和异常检测
- 实时数据看板

### Phase 4 — 协作与生态

- 多人协作
- 模板市场
- 嵌入式组件输出

## 竞品差异

|  | ChatGPT + Code Interpreter | Tableau / Power BI | ICanDraw |
|---|---|---|---|
| AI 原生 | ✅ 但输出是静态图片 | ❌ AI 是辅助 | ✅ AI 驱动，输出可交互 |
| 可交互图表 | ❌ | ✅ | ✅ |
| 手绘图解 | ❌ | ❌ | ✅ |
| 混合排版 | ❌ | ❌ | ✅ |
| 对话迭代 | ✅ 但无法增量修改 | ❌ | ✅ 增量修改画布 |

核心壁垒：**AI 输出的不是死图片，而是可交互、可编辑、可持续迭代的活画布。**
