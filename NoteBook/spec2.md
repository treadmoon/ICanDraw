# ICanDraw — AI 驱动的数据数据叙事智能画布 (优化版 v2.0)

## 产品定位 (重塑)

**原定位**：用自然语言对话，AI 自动生成数据可视化故事板——融合交互式图表与手绘风格图解的智能画布。
**新定位**：ICanDraw 不只是一个绘图工具，更是一个**“数据叙事伴侣”**。它通过 AI 消除“数据洞察”与“视觉传达”之间的缝隙。用户提供原始数据，AI 提供**可交互、可感知、带情绪的视觉故事**。

核心逻辑：用户提供上下文（原始数据/文档/对话） → AI 理解意图与场景 → 自动生成“基础骨架图表 + 情绪化手绘批注 + 智能文字摘要”的组合画面，并支持“Human-in-the-loop”的滑爽微调体验。

## 目标用户与核心场景

*   **产品经理 / 创业团队**：快速构思商业故事，解释复杂架构，需要高自由度和“草稿感”以降低决策压力和试错成本。
*   **数据分析师 / 运营**：周报/月报汇报，需要精准 ECharts 图表，并结合突出重点的手绘指示线来讲述数据背后的“Why”。
*   **技术架构师**：系统架构梳理，结合分层布局与依赖连线，把复杂逻辑变直观。

### 场景流（Scenes 预设框架）
进入产品时，用户不仅可以面对空白画布，还可以选择“场景预设”，AI 将根据场景采用不同的布局策略和生成风格：
*   **[工作复盘/汇报]**：为主流商业场景设计。默认采用专业/商务图表风格 + AI 自动查找极值、异常值并添加手绘标记。
*   **[原型/产品构思]**：全手绘风格 (充分发挥 Excalidraw UI的亲和力) + 逻辑箭头 + 占位符图表（先定逻辑再填数据）。
*   **[复杂系统/架构]**：分层网络布局 + 依赖关系自动梳理连线。

## AI 驱动的无缝协作体验 (UI/UX 核心)

### 1. 从“侧边栏对话”升级为“沉浸式画板生成” (Magic Transition)
*   **交互**：用户在全局指令栏输入描述。
*   **动效**：指令文字在回车后，化为粒子或卡片“飞”向主画布的预设锚点，逐层渲染为数据图表和周边元素。
*   **智能增强**：AI 画出柱状图后，不仅呈现图表，还会自动用**虚线圈出**最低的柱子，并用手写字体拉出一个箭头：“这周大盘流量下跌的遗留影响？”。

### 2. 画笔即指令 (Ink as Command)
弱化每次都要通过文字 Prompt 控制的方式，实现真正的“所见即所得”的协作：
*   **圈选触发**：用户拿起“智能画笔”（Smart Pen），在已有折线图的某阶段画个圈。
*   **即时上下文**：系统感知选中区域的数据特征，在圆圈旁弹出极简 Context 菜单：`[预测未来趋势] [与竞品对比] [深入归因分析] [换种展示图表]`。
*   **心智模型**：让用户感觉像在白板前与一位懂业务的数据专家肩并肩指点江山，而不仅仅是“发号施令”。

### 3. 一键风格漫游 (Style Morphing)
灵活适应不同社交和会议语境。
*   **微件控制**：UI 顶部提供全局滑块 `[严谨商务 (Standard)] ────── ● ────── [发散脑暴 (Sketch)]`
*   **丝滑变换**：滑动滑块时，标准的 ECharts 边框和线条开始产生微弱的“抖动（Jiggle）”，逐渐变为手绘风格，字体从严肃的 Sans-serif 切换为手写体，色彩也从严谨色板向粉笔/蜡笔质感过渡。

## 技术方案核心策略

1.  **坚持结构化双擎输出**：坚决摒弃“生成图片”的思路。LLM 的输出永远是可结构化的代码和数据定义 `JSON (ECharts) + JSON (Excalidraw)`，确保内容的绝对可逆和可二次干预。
2.  **Anchor-based 锚点布局引擎**：解决多元素画布排版随机、互相遮挡的痛点。AI 在作图时，会优先确立 1-2 张核心图表作为“重力锚点（Anchor）”，所有的手绘注释、贴纸、分析结论将围绕锚点进行弹性（Flexbox/碰撞检测）定位。
3.  **Privacy/Local First (隐私与本地优先)**：针对 B 端用户极其敏感的数据安全诉求。用户的 CSV/Excel 文件数据解析与聚合统计（如求和、均值计算），尽可能放在浏览器端基于 WASM(如 DuckDB-WASM) 或 Web Worker 处理。请求 LLM 时，仅发送脱敏的统计结论或 Schema，而不是全量数据，消除安全顾虑。

## 分阶段执行演进路线

### Phase 1: “Wow” 体验起点验证 (MVP, 4周)
**核心目标**：实现“图表 + 情绪批注”的无缝融合输出，打通产品主逻辑。
*   跑通大前端架构：ECharts 图表实例与 Excalidraw 画布层级的完美结合（允许拖拽、缩放图表框）。
*   核心对话链路：自然语言输入 → AI 解析指令生成 ECharts数据结构 → AI 提取数据特征并生成至少一条有意义的手绘批注。
*   支持单文件基础解析（如 CSV 拖入画布）。

### Phase 2: “局部协作”与场景流建立 (4周)
**核心目标**：落地上下文敏感交互，减少用户的 Prompt 编写压力。
*   上线“智能画笔 / 悬浮 AI 工具箱”：支持用户点击已生成的图表或画布元素，唤出 AI 面板执行“增量指令”（更换维度、补充注解、修改样式）。
*   场景预设选择（工作复盘 / 构思维度切换）。
*   上线 1.0 版锚点排版算法，保证增量生成内容的排版可用性。

### Phase 3: 数据智能大脑升级与连接 (4-6周)
**核心目标**：从静态生成走向动态实时监控与深度分析。
*   完善浏览器端 WASM 的高效数据处理引擎（支持过百万行级别的纯前端聚合分析）。
*   支持外部连接器（MySQL / 第三方 API 实时数据源接入），将死图表变为活看板。
*   强化 AI Root-cause 分析引擎，能对多张关联表的数据进行自动探查和可视化展示。

### Phase 4: 多人协作与无边界生态
**核心目标**：打造协作网络和内容池。
*   基于 CRDT(如 Yjs) 实现多人的白板实时协同编排（包括两人同时修改一张表的参数）。
*   内置社区模版市场：用户可以将自己的某次汇报布局定义为模板，一键复用给团队其他成员。
*   输出与集成：提供嵌入代码 (iframe/JS SDK)，支持将 ICanDraw 制作的交互画布直接嵌入至 Notion、飞书文档、内部 CMS 系统中，突破信息孤岛。

## 竞品防御壁垒与终局优势

相比于传统的 BI 系统（Tableau、Power BI）或者现有的 AI 绘图/代码生成工具（Napkin.ai、v0、ChatGPT Data Mode），ICanDraw 具有极度差异化的护城河：

| 维度 | AI 制图 (如 v0, Napkin) | 传统重型 BI | ICanDraw (优化版) |
| :--- | :--- | :--- | :--- |
| **可编辑自由度** | 依赖 Prompt 迭代，局部修改困难 | 必须依靠熟练配置，学习曲线陡峭 | **基于 Excalidraw，万物皆可直观拖拽拖拽与微调** |
| **数据深度与交互** | 常常仅为静态结果展示/不可选数据 | 深度分析强，但展现形式死板 | **保留 ECharts 在浏览器侧的强大交互（缩放、筛选、Tooltip）** |
| **视觉沟通质感** | 工业化、现代扁平 | 严肃拘谨，适合最终财报 | **独有的“手绘与专业混搭”，极度契合沟通探讨的心智预期** |
| **人机协作模式** | Bot-centric（我发指令你生成全屏幕）| User-centric（完全靠人鼠标点击）| **Copilot-centric（用户提供数据，AI 给草图，用户随手按需指点江山更改）** |

**一句话总结**：ICanDraw 卖的不是图表能力本身，而是团队内部开会时那块**“既有准确数据支撑，又能随意涂鸦、激发灵感的第一块白板”**。
