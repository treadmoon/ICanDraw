# ICanDraw — AI 驱动的数据叙事智能画布 (v3.0)

## 产品定位

ICanDraw 是一个**"数据叙事伴侣"**——通过 AI 消除"数据洞察"与"视觉传达"之间的缝隙。用户提供原始数据，AI 提供**可交互、可感知、带情绪的视觉故事**。

核心逻辑：用户提供上下文（原始数据/文档/对话） → AI 理解意图与场景 → 自动生成"基础骨架图表 + 情绪化手绘批注 + 智能文字摘要"的组合画面，并支持"Human-in-the-loop"的滑爽微调体验。

**一句话总结**：ICanDraw 卖的不是图表能力本身，而是团队内部开会时那块**"既有准确数据支撑，又能随意涂鸦、激发灵感的第一块白板"**。

## 目标用户与核心场景

- **产品经理 / 创业团队**：快速构思商业故事，解释复杂架构，需要高自由度和"草稿感"以降低决策压力和试错成本。
- **数据分析师 / 运营**：周报/月报汇报，需要精准 ECharts 图表，并结合突出重点的手绘指示线来讲述数据背后的"Why"。
- **技术架构师**：系统架构梳理，结合分层布局与依赖连线，把复杂逻辑变直观。

### 场景预设框架（Scenes）

进入产品时，用户不仅可以面对空白画布，还可以选择"场景预设"，AI 将根据场景采用不同的布局策略和生成风格：

- **[工作复盘/汇报]**：专业/商务图表风格 + AI 自动查找极值、异常值并添加手绘标记。
- **[原型/产品构思]**：全手绘风格（充分发挥 Excalidraw 的亲和力）+ 逻辑箭头 + 占位符图表（先定逻辑再填数据）。
- **[复杂系统/架构]**：分层网络布局 + 依赖关系自动梳理连线。

## AI 驱动的交互模式

### 1. 沉浸式画板生成（Magic Transition）

- 用户在全局指令栏输入描述
- 指令文字回车后，化为粒子/卡片"飞"向主画布的预设锚点，逐层渲染为数据图表和周边元素
- AI 画出柱状图后，不仅呈现图表，还会自动用虚线圈出最低的柱子，并用手写字体拉出箭头标注洞察

### 2. 画笔即指令（Ink as Command）

弱化纯文字 Prompt 控制，实现"所见即所得"的协作：

- **Phase A — 点击触发（初期）**：用户点击已有图表元素，在旁边弹出极简 Context 菜单：`[预测趋势] [竞品对比] [归因分析] [换图表类型]`
- **Phase B — 圈选触发（后期）**：用户拿起"智能画笔"，在折线图某阶段画个圈，系统感知选中区域的数据特征，弹出上下文操作
- **心智模型**：让用户感觉像在白板前与一位懂业务的数据专家肩并肩指点江山

### 3. 风格漫游（Style Morphing）

- UI 顶部全局滑块：`[严谨商务 (Standard)] ────── ● ────── [发散脑暴 (Sketch)]`
- 滑动时，ECharts 边框和线条产生"抖动（Jiggle）"过渡为手绘风格，字体从 Sans-serif 切换为手写体，色彩从严谨色板向粉笔/蜡笔质感过渡

### 4. AI 主动洞察

- AI 不只被动执行，还会主动发现数据中的异常、趋势、关联
- "我注意到你的用户留存率在 3 月有明显下降，要我深入分析吗？"

## 技术方案核心策略

1. **结构化双引擎输出**：LLM 输出永远是 `JSON (ECharts option) + JSON (Excalidraw elements)`，确保内容可逆、可二次干预，坚决摒弃生成图片的思路。

2. **Anchor-based 锚点布局引擎**：AI 作图时优先确立 1-2 张核心图表作为"重力锚点"，所有手绘注释、贴纸、分析结论围绕锚点进行弹性定位（Flexbox + 碰撞检测），解决多元素排版混乱的痛点。

3. **Privacy / Local First**：用户的 CSV/Excel 数据解析与聚合统计在浏览器端基于 DuckDB-WASM / Web Worker 处理。请求 LLM 时仅发送脱敏的统计结论或 Schema，不发全量数据。

4. **渲染层级方案**：ECharts 通过 SVG ForeignObject 嵌入 Excalidraw 画布，需做好跨浏览器兼容性测试（Safari 的 ForeignObject 支持有坑），备选方案为 Canvas overlay + 事件代理。

## 分阶段执行路线

### Phase 1：核心链路验证（MVP, 5-6 周）

**目标**：实现"图表 + 情绪批注"的融合输出，打通产品主逻辑。

- 大前端架构搭建：ECharts 实例与 Excalidraw 画布的层级融合（拖拽、缩放图表框）
- 核心对话链路：自然语言 → AI 生成 ECharts option → AI 提取数据特征生成手绘批注
- CSV 文件拖入画布解析
- DuckDB-WASM 本地数据处理集成

### Phase 2：上下文敏感交互（4 周）

**目标**：落地局部协作交互，减少用户 Prompt 编写压力。

- 点击图表元素唤出 AI 面板，执行增量指令（换维度、补注解、改样式）
- 场景预设选择（工作复盘 / 产品构思 / 架构梳理）
- 锚点排版算法 1.0，保证增量生成内容排版可用
- 风格滑块 + Jiggle 过渡动效

### Phase 3：数据智能升级（4-6 周）

**目标**：从静态生成走向动态实时与深度分析。

- WASM 数据引擎优化（支持百万行级纯前端聚合）
- 外部数据源连接器（MySQL / REST API / Google Sheets）
- AI Root-cause 分析引擎：多表关联自动探查与可视化
- 智能画笔圈选触发（Phase B 的 Ink as Command）

### Phase 4：协作与生态

**目标**：打造协作网络和内容池。

- 基于 Yjs (CRDT) 的多人实时协同
- 社区模板市场：用户可将汇报布局定义为模板，一键复用
- 嵌入式输出：iframe / JS SDK，支持嵌入 Notion、飞书文档、内部 CMS
- 演示模式（类 PPT 翻页浏览）

## 竞品差异与防御壁垒

| 维度 | AI 制图 (v0, Napkin) | 传统 BI (Tableau, PowerBI) | 协作白板 (Miro + AI) | ICanDraw |
| :--- | :--- | :--- | :--- | :--- |
| **可编辑自由度** | 依赖 Prompt 迭代，局部修改困难 | 配置驱动，学习曲线陡峭 | 自由度高但无数据图表能力 | **万物皆可拖拽微调** |
| **数据深度与交互** | 常为静态结果 | 深度分析强，展现形式死板 | 无数据分析能力 | **ECharts 浏览器侧强交互** |
| **视觉沟通质感** | 工业化扁平 | 严肃拘谨 | 手绘风格但无数据支撑 | **手绘与专业混搭** |
| **人机协作模式** | Bot-centric | User-centric | User + AI 辅助 | **Copilot-centric** |
| **数据隐私** | 数据上传云端 | 私有部署成本高 | 数据上传云端 | **Local First，WASM 本地处理** |
